# 学习规则

定义 Skill 如何学习和改进的规则。

## 用户偏好学习

### 触发条件
- 用户在对话中表达格式/风格偏好
- 用户修正生成内容后询问是否记住

### 学习流程
```
用户修改生成内容 / 表达偏好
    ↓
询问：是否记住这个偏好？
    ↓
是 → memory_manager.py set-pref 写入 user-preferences.json
否 → 仅本次应用
```

### 可学习的偏好

| 类型 | 示例 |
|------|------|
| 用例标题风格 | "输入X，结果Y" 格式 |
| 步骤粒度 | 操作步骤合并 or 拆分 |
| 模块层级深度 | 最多 3 层 / 最多 5 层 |
| 预期结果风格 | 每步一条 / 汇总写 |
| 标签默认值 | 每次默认 APP |

## 术语学习

### 触发条件
- 解析需求文档时发现专业术语
- 用户在对话中解释术语或修正叫法

### 识别模式
- 括号解释：`GMV（商品交易总额）`
- 冒号解释：`SKU：库存单位`
- 首字母大写缩写：自动检测
- 引号定义：`"SKU" 指库存单位`

### 学习内容
- **领域术语**：专业名词及含义、缩写全称
- **模块缩写**：模块名称 → 缩写映射，用于生成用例编号

### 存储格式（terminology.json 示例）
```json
{
  "GMV": "商品交易总额",
  "SKU": "库存单位",
  "LGN": "登录模块"
}
```

## 用户反馈学习

### 反馈类型

| 类型 | 示例 | 处理方式 |
|------|------|---------|
| 纠正错误 | "这条用例逻辑不对" | 重新生成 + 记录歧义模式到 `ambiguity-decisions.json` |
| 删减用例 | "P2 太多了" | 询问"是否调整优先级分布作为默认？"→ 写入 `user-preferences.json` |
| 补充场景 | "还缺并发场景" | 增补 + 记录为常见遗漏到 `generation-history.json` |
| 修改标签 | "这个应该是 APP" | 原样更新 `default_tag` |
| 修改术语 | "这里应该叫 XXX" | 询问"是否记住？"→ 写入 `terminology.json` |
| 调整步骤 | "步骤太细了，合并一下" | 询问"是否记住这个粒度偏好？"→ 写入 `user-preferences.json` |

### 反馈触发时机

生成完成后，用户对结果的任何修正建议都应触发学习流程（Phase 5）。AI 必须：
1. 先执行用户的修正请求
2. 再询问是否将该偏好/决策持久化
3. 用户同意后写入对应 `.memory/` 文件

## 持续改进

### 生成质量
1. **分析历史记录**：统计常见模块、识别重复遗漏的场景类型
2. **优化场景覆盖**：确保基本流 / 备选流 / 异常流 / 边界值均有覆盖
3. **复用模块结构**：相似模块共享测试模式，减少重复设计

### 效率
1. **缓存常用术语**：跨会话复用 terminology.json，避免重复提取
2. **预测用户偏好**：基于 user-preferences.json 减少重复确认

## 记忆管理命令

| 用户指令 | 对应脚本操作 | 说明 |
|---------|------------|------|
| `更新术语表` | `memory_manager.py update --project $SKILL_ROOT` | 重新扫描 terminology.json |
| `清除记忆` | `memory_manager.py clear --project $SKILL_ROOT` | 重置所有学习数据 |
| `显示学习记录` | `memory_manager.py read --project $SKILL_ROOT` | 查看历史生成与偏好 |
| `查找歧义` | `memory_manager.py find-ambiguity --project $SKILL_ROOT --keyword <词>` | 查询已记录的歧义处理 |
