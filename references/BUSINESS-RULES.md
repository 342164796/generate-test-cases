> **使用说明**：在此填写你的项目业务背景，帮助 AI 更准确地理解需求意图、判断测试边界与优先级。
>
> 建议包含以下信息：
> - 产品定位（面向什么行业/用户群体）
> - 主要功能模块列表
> - 端侧划分（B 端/C 端/小程序等及其目标用户）

**示例格式**：

```
本项目为面向 [行业] 的 [产品类型]，覆盖 [核心场景]，主要功能模块包括：

| 模块 | 说明 |
|------|------|
| 模块A | 功能1、功能2 |
| 模块B | 功能3、功能4 |

端侧划分：
- B 端（Web + APP）：内部运营人员
- C 端（小程序）：end user
```

## 列表

> 适用：任何带有数据列表、表格的功能模块

| 场景 | 测试要点 |
|------|---------|
| 空列表 | 无数据时显示空状态占位图/文案，不报错、不白屏 |
| 大数据量 | 数据量达上限时页面不卡顿，滚动/翻页正常 |
| 分页 | 首页、末页、中间页切换正确；页码超出范围给出提示 |
| 加载更多/无限滚动 | 滚动到底触发加载，loading 态，无更多数据时提示 |
| 排序 | 默认排序符合需求；切换字段/方向后列表正确刷新 |
| 筛选 | 多个筛选条件组合有结果、无结果；清空后恢复全量 |
| 携带筛选翻页 | 翻页后筛选条件继续生效，数据仍符合筛选条件 |
| 搜索 | 名称搜索、模糊搜索结果准确；无结果时给出提示 |
| 多选操作 | 全选/取消全选；跨页选择保留；批量操作后列表刷新 |
| 文案溢出 | 表格内容超出一行时正常换行或省略，不破坏布局 |
| 时间筛选-最大跨度 | 选择最早到最晚时间，系统能正常处理不报错 |
| 时间筛选-边界合理性 | 最早可选时间、最晚可选时间符合业务规则 |
| 时间筛选-快捷选项 | 本周/上周/本月/上月等快捷选项计算结果正确 |
| 业务逻辑验证 | 状态/开关控制的展示逻辑、统计计算逻辑（如总销量规则）准确 |
| 导出回归 | 列表结构变更后，导出内容与列表字段保持一致 |
| 异常 | 网络异常时给出加载失败提示，不白屏 |

## 按钮

> 适用：任何触发操作的按钮、链接、图标按钮

### 通用规则

| 场景 | 测试要点 |
|------|---------|
| 防重复点击（正常网络） | 点击后立即进入 loading/禁用态，请求完成前不可再次触发 |
| 防重复点击（弱网） | 弱网下连续点击，不会重复生成/更新数据 |
| 禁用态 | 条件不满足时按钮置灰，点击无响应 |
| 加载态 | 异步操作期间显示 loading，完成后恢复 |

### 提交/确定/保存按钮（必须生成 4 条用例）

| 用例 | 测试要点 |
|------|---------|
| 1. 校验拦截 | 必填项为空时点击，展示错误提示且不提交 |
| 2. 提交成功 | 所有字段合法时点击，数据正确生成/保存 |
| 3. 正常网络防重 | 快速连续点击，按钮进入禁用/loading，不重复提交 |
| 4. 弱网防重 | 弱网下连续点击，不会重复生成/更新数据 |

### 取消按钮（必须生成 2 条用例）

| 用例 | 测试要点 |
|------|---------|
| 1. 二次确认 | 点击取消弹出确认提示（如"未保存内容将丢失，是否确认？"） |
| 2. 数据不保存 | 确认取消后，弹窗/页面关闭，已填数据未被保存 |

## 详情页/展示类

> 适用：查看详情、弹窗展示等只读展示场景

### 字段展示

- 需求中列出的**每个字段**生成 1 条独立用例，验证内容与数据源一致
- 有多个枚举值/状态的字段，**合并为 1 条用例**，steps 中逐一覆盖各状态
- **禁止验证 UI 样式**（颜色、字体、间距、边框等），只验证内容和数据准确性

### 特殊展示场景

| 场景 | 测试要点 |
|------|---------|
| 条件展示字段 | 某些字段仅在特定条件下显示，验证显示/隐藏逻辑 |
| 多值字段 | 数量角标准确；hover 展开完整列表；超量时省略+hover 查看全部 |
| 长文本展示 | 多行正确换行；纯英文长单词不溢出；emoji/特殊字符正常显示 |
| 字段为空 | 字段无值时展示"-"或隐藏，不报错 |
| 图片展示 | 图片正常加载；加载失败显示占位图；支持点击放大预览 |

### 详情页内的表格

| 场景 | 测试要点 |
|------|---------|
| 表头完整 | 所有表头字段正确展示 |
| 列数据准确 | 每列数据与实际匹配 |
| 条件展示按钮 | 操作按钮仅在满足条件的行显示（如"取消"仅对特定状态行显示） |
| 空表格 | 无数据时展示空状态提示 |

## 多入口场景

> 适用：同一个操作/结果可通过多条路径达成（如学员通过多种方式入班）

- **相同验证目标的多入口：合并为 1 条用例**，steps 中逐一列出每个具体入口
- **不同验证目标的入口：拆分为多条用例**
- steps 必须写明每个入口的具体操作，禁止使用"通过多种方式"等模糊描述
- expected_result 与 steps 一一对应，数量相等
