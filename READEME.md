# 使用说明

## 简介

本工具是一个**自主学习型测试文档生成器**，通过 GitHub Copilot 与 Claude 驱动，从 Markdown 需求文档自动生成专业测试用例，并导出为 XMind 文件。核心能力：

- 4 种测试设计方法（EP/BVA/ST/EG）系统覆盖
- 需求追溯矩阵 + 覆盖率自动计算
- 持久化记忆，越用越准
- 质量自检门禁，生成前拦截问题

---

## 快速开始

### 第一步：写入需求

将需求内容写入 `requirements/requirement.md`（支持多个 `.md` 文件）：

```markdown
# 用户登录功能

## 需求描述
用户通过账号和密码登录系统。
...
```

也可以在 `requirements/` 目录中放入截图（`.png`、`.jpg` 等），工具会一并识别。

## 从图片生成测试用例

### 支持的使用场景

不是所有需求都有文字描述。很多时候需求来源是：

- **UI 设计稿**（Figma/蓝湖截图）
- **原型图**（Axure/墨刀截图）
- **流程图**（业务流程、状态机截图）
- **已有系统截图**（用于补充测试不完整的旧功能）
- **PRD 中的图表**（表格、规则说明截图）

这些图片可以**单独使用**，也可以**配合文字需求一起使用**（推荐）。

### 如何使用

将图片直接放入 `requirements/` 目录，与 `.md` 文件并列：

```
requirements/
├── requirement.md        ← 文字需求（可选）
├── login-ui.png          ← 登录页设计稿
├── checkout-flow.jpg     ← 结账流程图
└── rules-table.png       ← 规则说明截图
```

然后和往常一样触发生成即可：

```
生成测试用例
```

工具会**自动读取目录下所有图片**，与文字需求合并后一起分析，无需任何额外配置。

### 支持的图片格式

`.png` / `.jpg` / `.jpeg` / `.gif` / `.webp` / `.bmp` / `.svg`

### 工具从图片中识别什么

| 图片类型 | 工具识别内容 | 生成的用例方向 |
|---------|------------|-------------|
| UI 设计稿 | 页面元素、输入框、按钮、状态、提示文案 | 表单验证、交互反馈、空态/错误态展示 |
| 流程图 | 分支条件、步骤顺序、异常路径 | 基本流、备选流、异常流（场景法 ST） |
| 原型图 | 页面跳转关系、弹窗触发条件 | 页面间跳转用例、弹窗交互用例 |
| 规则表格截图 | 枚举值、条件组合、限制规则 | 等价类（EP）+ 边界值（BVA）用例 |

### 图文结合的最佳实践

单纯图片能生成用例，但**图 + 文字结合效果最好**：

- 文字需求：描述业务规则、验收标准、数据格式
- 图片：直观展示 UI 状态、流程分支、数据关系

```
requirements/
├── requirement.md        ← 写业务规则和验收条件
└── ui-design.png         ← 放对应的设计稿截图
```

工具会自动将两者交叉对照，补全单独依赖任何一方都可能遗漏的用例。

### 注意事项

- 图片文件名建议有语义（如 `login-page.png`），便于工具在解析摘要中显示来源
- 图片分辨率过低或内容模糊时，识别准确率会下降，建议使用清晰截图
- 纯图片模式（无 `.md` 文件）可以使用，但建议至少在 `requirement.md` 中写几行背景说明，帮助工具理解业务上下文

### 第二步：触发生成

在 Copilot Chat 中输入任意触发词：

```
生成测试用例
```

```
根据需求生成测试
```

```
分析这个 PRD，生成测试
```

### 第三步：按提示操作

工具采用**快速模式**，大部分步骤自动执行，仅在以下两个检查点暂停：

| 检查点 | 发生时机 | 需要你做什么 |
|--------|---------|-------------|
| **解析确认** | 解析完成后 | 确认识别到的模块和需求数量；首次使用时选择适用端（PC/APP 等） |
| **生成预览** | 生成前 | 确认用例数量和质量统计，回车继续 |

### 第四步：获取结果

生成完成后，XMind 文件保存在：

```
test-docs/testcases_<时间戳>.xmind
```

用 XMind 应用打开即可查看完整测试用例树。

```
generate-test-cases/
├── requirements/          ← 📝 将需求文档放在这里
│   └── requirement.md
├── test-docs/             ← 📦 生成的 XMind 文件输出到这里
├── tmp/                   ← 🗂  中间 JSON 文件（自动管理）
├── scripts/               ← 工具脚本（无需手动操作）
│   ├── generate_xmind.py
│   └── memory_manager.py
├── references/            ← 规范文档（供 AI 参考，无需修改）
├── .memory/               ← 持久化记忆（自动生成，勿手动修改）
├── SKILL.md               ← Skill 定义文件
└── CLAUDE.md              ← Claude CLI 入口
```

## 适用端（标签）说明

工具会在首次生成时询问测试用例适用的端，可选：

| 编号 | 选项 |
|------|------|
| 1 | PC |
| 2 | APP |
| 3 | C端 |
| 4 | PC, APP |
| 5 | 小程序 |
| 6 | 其他（自定义） |

你的选择会被记住，下次生成同类需求时自动使用，无需重复选择。

**提示**：若需求文档中已写明适用端（如 `tag只适用于PC端`），工具会直接读取，不会再询问。

## 记忆系统

首次运行后，项目根目录会自动生成 `.memory/` 文件夹：

```
.memory/
├── user-preferences.json     # 你的偏好（标签、步骤粒度等）
├── terminology.json          # 领域术语库
├── generation-history.json   # 历史生成记录
└── ambiguity-decisions.json  # 歧义处理决策
```

**记忆的作用**：
- 自动复用上次的标签选择
- 相同类型的歧义不再重复询问
- 历史中的常见遗漏场景自动补入新一轮生成

### 记忆管理命令

在 Copilot Chat 中直接输入：

```
查看偏好          # 查看当前保存的用户偏好
更新术语表        # 手动维护领域术语
清除记忆          # 重置所有记忆（慎用）
```

## 反馈与学习

生成后，你可以对结果直接提出修改意见，工具会自动学习：

| 你说的 | 工具做的 |
|--------|---------|
| "这条用例逻辑不对" | 重新生成 + 记录歧义模式 |
| "P2 太多了" | 询问是否调整默认优先级分布 |
| "还缺并发场景" | 增补用例 + 记录为常见遗漏 |
| "这里应该叫 XXX" | 询问是否写入术语表 |
| "步骤太细，合并一下" | 询问是否记住粒度偏好 |

## 生成质量说明

每次生成前，工具会自动进行以下 6 项质量检查，任一不通过则先修正再输出：

- 需求覆盖率 ≥ 95%
- P0 占比 10–15%，P1 占比 30–40%
- 每条需求至少关联 1 种设计方法
- 无需求文档之外编造的场景
- 无语义重复用例
- 优先级分布合理

## 常见问题

**Q: 支持多个需求文件吗？**  
A: 支持。将多个 `.md` 文件放在 `requirements/` 目录下，工具会按文件名排序后合并处理。

**Q: 能否在需求文档里放截图？**  
A: 支持。将 `.png`、`.jpg`、`.jpeg`、`.gif`、`.webp`、`.bmp`、`.svg` 放在 `requirements/` 目录下即可，无需额外配置。工具支持 UI 设计稿、流程图、原型图、规则表格截图等多种图片类型，详见「从图片生成测试用例」章节。

**Q: 生成的 XMind 文件打开是空的？**  
A: 确认 Python 3 已安装，且 `scripts/generate_xmind.py` 的依赖已就绪。可以在终端运行 `python3 --version` 检查。

**Q: 如何重新生成同一份需求？**  
A: 直接再次触发"生成测试用例"，会以新时间戳生成新文件，不会覆盖历史结果。

**Q: 怎么让工具记住我的偏好？**  
A: 在反馈阶段直接说明（如"步骤细一点"），工具会询问是否持久化，确认后下次自动生效。
